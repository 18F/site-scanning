Scans

There are a few ways to dig into the data. The Federal Website Scanner offers different types of scans which are described below. Browse the results of existing scans or customize the search to meet your needs. Explore these use cases to see how our partners are using the Federal Website Scanner.

File scanner

The file scanner checks for the presence or absence of specific files at specific locations including /code.json, /data.json, /data, /developer, /digitalstrategy, /open, /privacy, /robots.txt, and /sitemap.xml.

The file scanner checks for the 200 server response code from the domain's specific URL. The 200 server response code indicates that the file exists at the specific location and loads successfully.

The presence of these files at these specific locations is an indicator that the website is aligned with best practices as established by the 21st Century IDEA and other laws, policies, and regulations for federal websites.

USWDS scanner

The U.S. Web Design System (USWDS) is a design system for U.S. government websites that makes it easier to build accessible, mobile-friendly sites.

The USWDS scanner checks each domain for the presence and version of USWDS code. The scanner looks for specific code in the domain's CSS and HTML files, which fonts are in use, and and other USWDS indicators.

The presence of the latest version of the USWDS on a website is an indicator that the site is accessible and mobile-friendly.
